# ZenPulse: AI Meditation App

**Проект:** прототип мобильного приложения для медитаций с AI-генерацией аффирмаций.  
**Стек:** React Native + Expo, TypeScript, NativeWind, React Navigation.  
**Цель:** демонстрация навыков быстрой сборки мобильных интерфейсов, работы с подписками и интеграции AI.

## Навигация и логика подписки

- **Навигация:** реализована через React Navigation Native Stack.
- **Экран Paywall:** главный экран для новых пользователей, демонстрирует преимущества Premium-подписки.
- **Экран Meditation:** основной контент, карточки с медитациями.  
  Заблокированные карточки ведут обратно на Paywall, если `isSubscribed = false`.
- **Экран Mood (AI Настрой дня):** отдельный экран, на котором пользователь выбирает настроение (3 смайлика) и генерирует аффирмацию через AI.

---

## Мобильная специфика и адаптивность

- Использован **SafeAreaView** на всех экранах для корректного отображения на устройствах с вырезами (“челка”) и полосами состояния.
- **ScrollView / FlatList** применены для вертикального контента и списков.
- **Responsive layout:** проверено на iPhone SE (маленький экран) и iPhone 14 Pro Max (большой экран).  
- Использованы **адаптивные отступы и размеры** через NativeWind (Tailwind для React Native):
  - горизонтальные padding: 20–24
  - секции: 24–32
  - карточки медитаций: 16px spacing
- Карточки и кнопки масштабируются под разные размеры экранов; текст не выходит за границы.

---

## Интеграция AI

- Для генерации аффирмаций используется функция `generateAffirmation(mood)`.  
- MoodSelector и AffirmationBox — отдельные компоненты для удобной интеграции AI-контента.
- Все AI-генерации протестированы с мок-данными, чтобы UI оставался стабильным при отсутствии сети.

## Контрольный вопрос

**Вопрос:** «С какими специфическими проблемами мобильной верстки ИИ справляется хуже всего и как ты контролировал его работу, чтобы приложение не сломалось на маленьких экранах (iPhone SE vs Pro Max)?»

**Ответ:**

1. **Проблемы, с которыми ИИ часто справляется хуже всего:**
   - Некорректные отступы при SafeArea и notch.
   - Элементы выходят за границы экрана на маленьких устройствах.
   - Тексты не масштабируются и могут обрезаться.
   - Кнопки слишком маленькие или тесно расположены.
   - Карточки FlatList/ScrollView иногда генерируются без адаптивной ширины и height.

2. **Как я контролировал:**
   - Всегда проверял макеты на **iPhone SE и iPhone 14 Pro Max** в эмуляторе.
   - Добавлял **SafeAreaView**, ScrollView и адаптивные padding через NativeWind.
   - Использовал проверку визуальных границ и overflow текста.
   - В случае проблем формировал отдельные промпты к Gemini Flash 3 с уточнением **responsive fix**, чтобы AI корректировал layout без изменения логики.
   - Для кнопок и карточек применял минимальные размеры touch target (44px) и consistent spacing.
